## 项目概述

本项目是一个**港股/A股个股投资分析模板系统**，属于知识管理型项目，非传统软件工程。核心产出是一套标准化的投资分析框架和配套工具，帮助投资者进行深度个股研究。

### 项目定位

- **类型**: 投资分析模板 + 辅助工具脚本
- **目标用户**: 个人投资者（当前资金规模：十来万港币）
- **投资市场**: 港股（优先）、A股
- **投资流派**: 四流派体系（纯硬收息/价值发现/烟蒂股/关联方资源型）

### 核心文件

| 文件 | 用途 | 重要性 |
|------|------|--------|
| `个股分析标准模版.md` | 🔥 **唯一核心工作文件**，包含完整分析框架（V5.5.8） | 最高 |
| `README.md` | 项目入口、快速开始指南 | 高 |
| `CHANGELOG.md` | 版本更新日志 | 中 |
| `01-筛选框架/hk_stock_screener.py` | 港股自动筛选脚本 | 中 |
| `config/risk_management.yaml` | 风控规则配置 | 中 |

## 开发规范

### 版本管理（极其重要）

本项目采用**严格的版本管理制度**，版本号是核心标识：

```
版本格式: V{主版本}.{次版本}.{修订号}
当前版本: V5.5.8
```

#### 版本更新流程

1. **编辑当前文件**: 直接在 `个股分析标准模版.md` 上修改，不要新建文件
2. **更新版本号**: 同步更新以下位置的版本号：
   - 文档顶部标题
   - README.md
   - CHANGELOG.md
   - 所有脚本中的版本注释
3. **记录变更**: 在 CHANGELOG.md 添加变更记录
4. **归档旧版本**: 将旧版本复制到 `归档版本/` 目录

#### 文件命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 当前版本 | `个股分析标准模版.md` | （固定，始终是最新版） |
| 历史版本 | `个股分析标准模版_V{版本号}_{版本名}.md` | `个股分析标准模版_V5.5.6_实战机制深化版.md` |
| 提交记录 | `提交记录_V{版本号}.md` | `提交记录_V5.5.6.md` |
| **分析报告** | `{公司名称}_{股票代码}_投资分析报告.md` | `保利物业_06049_投资分析报告.md` |

> **AI助手注意**: 当用户要求生成投资分析报告时，必须在 `07-分析输出/` 目录下新建文件，使用 `{公司名称}_{股票代码}_投资分析报告.md` 格式命名。**禁止**创建"快速参考"类文件。

### 文档编写规范

#### 标签系统

在分析模板中，使用特定标签标记任务类型：

| 标签 | 含义 | AI角色 | 人工角色 |
|------|------|--------|----------|
| `[COMPUTABLE]` | 可计算指标 | 抓取数据、计算比率、生成表格 | 验证数据源、检查异常值 |
| `[LLM-REQUIRED]` | 需AI分析 | 文本分析、模式识别、趋势判断 | 复核逻辑、确认结论 |
| `[MANUAL-ONLY]` | 必须人工 | 提供计算框架 | 最终判断、决策执行 |
| `[CROSS-CHECK]` | 交叉验证 | 提供多源数据对比 | 确认差异原因 |
| `[TREND-ANALYSIS]` | 趋势分析 | 计算5年趋势、标记异常 | 判断趋势质量 |
| `[DATA-CONFIDENCE]` | 数据置信度 | 标记数据可信度等级 | 核实关键数据 |

#### 数据源分级

| 等级 | 数据源 | 使用限制 |
|------|--------|---------|
| **S级** | 公司年报、港交所披露易 | 可全额建仓 |
| **A级** | 公司官网、投资者关系 | 可正常分析 |
| **B级** | Bloomberg、Wind、东方财富 | 需与S级交叉验证 |
| **C级** | 财经媒体、研报、社区 | 仅供参考 |
| **D级** | AI估算、行业惯例推测 | **禁止用于投资决策** |

### 代码编写规范

#### Python脚本

1. **版本注释**: 每个脚本头部必须标注对应模板版本
   ```python
   """
   港股金龟筛选器 V1.2 (V5.5.7熊市+消除变量版)
   基于投资模板V5.5.7标准自动筛选港股标的
   """
   ```

2. **编码声明**: 必须包含 UTF-8 编码声明
   ```python
   # -*- coding: utf-8 -*-
   ```

3. **配置集中**: 筛选标准等配置使用字典常量，如 `SCREENING_CONFIG`

#### YAML配置

- 使用中文注释说明配置项
- 版本号必须与主模板一致
- 结构：仓位限制 → 止损规则 → 流动性限制 → 汇率风险 → 市场整体风控

### 投资哲学核心（从实战经验提炼）⭐⭐⭐⭐⭐

#### 1. 龟龟投资理论（防御型价值投资）

**理论渊源**：格雷厄姆-多德理论（70-80%）+ 港股/A股现状及个人会计学视角改良（20-30%）

**核心特征**：
- **防御性强**：市场下跌时回撤小
- **追求绝对防御能力**：牛市80%时间跑赢指数，熊市90%以上时间跑赢指数
- **收益结合回撤看**：注重赔率与胜率的综合预期值，而非单纯收益率

#### 2. 普通人假设（能力圈认知）

**承认自身能力平庸**：
- 视线仅限**1-2年**，无信息优势
- 回避预测大宗商品（油价、航运）等高波动变量
- **不做"大学生数学题"，只做简单确定的"小学生数学题"**

**实践要求**：
- 偏好商业模式稳定、偏刚性消费、日常必需的行业
- 不刻意逆向投资，而是因景气赛道估值过高，被迫选择被杀估值的"鬼故事"板块

#### 3. 反对"压票思维"（动态估值原则）

**定义**：不认同无论价格如何都长期持有某家公司

**操作原则**：
- **比较内在价值与市场报价**，弱化对3-5年后现金流的预测
- 当估值达公允值（如海底捞19倍PE），因**机会成本**卖出切换
- 当标的估值优势不再（如阿里现金储备从7400亿缩水至3000亿），果断止损

#### 4. 基金模式持仓（分散与集中平衡）

**持仓结构**：
- 高度分散，持仓约**10只个股**
- 单一个股仓位**不超过20%（通常为10%左右）**
- 类似基金模式，**不搞全副身家押注某几家公司**

**vs 模板原标准**：
- 模板：核心仓位10%、卫星5%、观察2%
- 实战优化：单只≤10%，更分散，降低单一标的非系统性风险

#### 5. 风险记忆（心理建设）

**记录历史最大回撤**：
- 经历过恒生指数从22000点跌至14000-15000点
- 曾浮亏60-70万
- 港股市场血腥，个股可能跌90%以上

**行为影响**：
- 始终保持提防心理
- 注重安全边际和低波动率
- 不因短期反弹而放松风控

#### 6. 最小单位交易（操作纪律）

**交易频率**：极少做T

**交易单位**：
- 多为**500-1000股最小单位交易**
- 不影响整体仓位
- 适用于微调持仓或试探性操作

## AI助手工作规范

### 生成分析报告命名规范

当用户要求生成个股投资分析报告时，**必须**遵守以下规范：

1. **输出目录**: `07-分析输出/`
2. **文件命名格式**: `{公司名称}_{股票代码}_投资分析报告.md`
3. **正确示例**:
   - `腾讯控股_00700_投资分析报告.md`
   - `阿里巴巴_09988_投资分析报告.md`
   - `美团_03690_投资分析报告.md`

4. **禁止事项**:
   - ❌ 不要创建 "快速参考"、"速览" 等辅助文件
   - ❌ 不要使用非标准命名格式
   - ❌ 不要输出到其他目录

### 分析报告自动更新到网页规范 ⭐

当生成新的投资分析报告后，**必须**自动更新 VitePress 侧边栏配置，确保新报告能在网页上显示。

#### 自动更新流程

```
生成新报告 → 更新 .vitepress/config.mjs → 推送到GitHub → Vercel自动部署
```

#### 具体操作步骤

**Step 1：生成报告后，立即更新侧边栏配置**

编辑 `.vitepress/config.mjs`，在 `07-分析输出` 章节添加新报告链接：

```javascript
{
  text: '📈 07-分析输出',
  collapsed: true,
  items: [
    { text: '保利物业_06049_投资分析报告', link: '/07-分析输出/保利物业_06049_投资分析报告' },
    { text: '天津发展_00882_投资分析报告', link: '/07-分析输出/天津发展_00882_投资分析报告' },
    { text: '中国民航信息网络_00696_投资分析报告', link: '/07-分析输出/中国民航信息网络_00696_投资分析报告' },
    { text: '金融街物业_01502_投资分析报告', link: '/07-分析输出/金融街物业_01502_投资分析报告' },
    // 新增报告在此添加
    { text: '{新公司名称}_{股票代码}_投资分析报告', link: '/07-分析输出/{文件名}' }
  ]
}
```

**Step 2：提交并推送**

```bash
git add .
git commit -m "新增{公司名称}分析报告，更新侧边栏配置"
git push
```

**Step 3：Vercel自动部署**

- Vercel 检测到 `main` 分支推送后自动重新构建
- 约 1-2 分钟后，新报告在网页侧边栏可见
