## 项目概述

本项目是一个**港股/A股个股投资分析模板系统**，属于知识管理型项目，非传统软件工程。核心产出是一套标准化的投资分析框架和配套工具，帮助投资者进行深度个股研究。

### 项目定位

- **类型**: 投资分析模板 + 辅助工具脚本
- **目标用户**: 个人投资者（当前资金规模：十来万港币）
- **投资市场**: 港股（优先）、A股
- **投资流派**: 四流派体系（纯硬收息/价值发现/烟蒂股/关联方资源型）

### 核心文件

| 文件 | 用途 | 重要性 |
|------|------|--------|
| `个股分析标准模版.md` | 🔥 **唯一核心工作文件**，包含完整分析框架（V5.5.7） | 最高 |
| `README.md` | 项目入口、快速开始指南 | 高 |
| `CHANGELOG.md` | 版本更新日志 | 中 |
| `01-筛选框架/hk_stock_screener.py` | 港股自动筛选脚本 | 中 |
| `config/risk_management.yaml` | 风控规则配置 | 中 |

## 开发规范

### 版本管理（极其重要）

本项目采用**严格的版本管理制度**，版本号是核心标识：

```
版本格式: V{主版本}.{次版本}.{修订号}
当前版本: V5.5.7
```

#### 版本更新流程

1. **编辑当前文件**: 直接在 `个股分析标准模版.md` 上修改，不要新建文件
2. **更新版本号**: 同步更新以下位置的版本号：
   - 文档顶部标题
   - README.md
   - CHANGELOG.md
   - 所有脚本中的版本注释
3. **记录变更**: 在 CHANGELOG.md 添加变更记录
4. **归档旧版本**: 将旧版本复制到 `归档版本/` 目录

#### 文件命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 当前版本 | `个股分析标准模版.md` | （固定，始终是最新版） |
| 历史版本 | `个股分析标准模版_V{版本号}_{版本名}.md` | `个股分析标准模版_V5.5.6_实战机制深化版.md` |
| 提交记录 | `提交记录_V{版本号}.md` | `提交记录_V5.5.6.md` |
| **分析报告** | `{公司名称}_{股票代码}_投资分析报告.md` | `保利物业_06049_投资分析报告.md` |

> **AI助手注意**: 当用户要求生成投资分析报告时，必须在 `07-分析输出/` 目录下新建文件，使用 `{公司名称}_{股票代码}_投资分析报告.md` 格式命名。**禁止**创建"快速参考"类文件。

### 文档编写规范

#### 标签系统

在分析模板中，使用特定标签标记任务类型：

| 标签 | 含义 | AI角色 | 人工角色 |
|------|------|--------|----------|
| `[COMPUTABLE]` | 可计算指标 | 抓取数据、计算比率、生成表格 | 验证数据源、检查异常值 |
| `[LLM-REQUIRED]` | 需AI分析 | 文本分析、模式识别、趋势判断 | 复核逻辑、确认结论 |
| `[MANUAL-ONLY]` | 必须人工 | 提供计算框架 | 最终判断、决策执行 |
| `[CROSS-CHECK]` | 交叉验证 | 提供多源数据对比 | 确认差异原因 |
| `[TREND-ANALYSIS]` | 趋势分析 | 计算5年趋势、标记异常 | 判断趋势质量 |
| `[DATA-CONFIDENCE]` | 数据置信度 | 标记数据可信度等级 | 核实关键数据 |

#### 数据源分级

| 等级 | 数据源 | 使用限制 |
|------|--------|----------|
| **S级** | 公司年报、港交所披露易 | 可全额建仓 |
| **A级** | 公司官网、投资者关系 | 可正常分析 |
| **B级** | Bloomberg、Wind、东方财富 | 需与S级交叉验证 |
| **C级** | 财经媒体、研报、社区 | 仅供参考 |
| **D级** | AI估算、行业惯例推测 | **禁止用于投资决策** |

### 代码编写规范

#### Python脚本

1. **版本注释**: 每个脚本头部必须标注对应模板版本
   ```python
   """
   港股金龟筛选器 V1.2 (V5.5.7熊市+消除变量版)
   基于投资模板V5.5.7标准自动筛选港股标的
   """
   ```

2. **编码声明**: 必须包含 UTF-8 编码声明
   ```python
   # -*- coding: utf-8 -*-
   ```

3. **配置集中**: 筛选标准等配置使用字典常量，如 `SCREENING_CONFIG`

#### YAML配置

- 使用中文注释说明配置项
- 版本号必须与主模板一致
- 结构：仓位限制 → 止损规则 → 流动性限制 → 汇率风险 → 市场整体风控

## 构建与运行

### 本地运行筛选脚本

```bash
# 进入筛选框架目录
cd 01-筛选框架

# 运行筛选器
python hk_stock_screener.py

# 输出：生成 hk_candidates_{timestamp}.csv 文件
```

### 安装依赖

```bash
pip install akshare pandas matplotlib yfinance
```

### GitHub Actions 自动运行

- **触发条件**: 
  - 定时：香港时间 16:30（UTC 08:30），工作日运行
  - 手动：可通过 workflow_dispatch 手动触发
  
- **运行步骤**:
  1. 检出代码
  2. 设置 Python 3.10 环境
  3. 安装依赖
  4. 运行追踪脚本
  5. 提交追踪数据到仓库
  6. 如有买入信号，创建 GitHub Issue 提醒

## 测试策略

本项目为知识型项目，**无传统单元测试**，采用以下质量保证机制：

### 数据核查机制

1. **强制性核查清单** (`02-数据清洗/02-1-强制性核查清单.md`)
2. **交叉验证**: 多数据源对比（Bloomberg/东方财富/港交所）
3. **数据错误作废机制**: 基于错误数据的分析必须作废

### 版本核查清单

每次分析前必须进行：
- [ ] 打开的文件名为 `个股分析标准模版.md`（不是归档版本）
- [ ] 文档顶部版本号与 README 一致
- [ ] 数据时点为最新财报

## 安全注意事项

### 投资安全

1. **免责声明**: 本模板仅为个人投资研究使用，不构成投资建议
2. **数据验证**: AI计算结果必须经过人工验证，未经核查的数据不得用于投资决策

## AI助手工作规范

### 生成分析报告命名规范

当用户要求生成个股投资分析报告时，**必须**遵守以下规范：

1. **输出目录**: `07-分析输出/`
2. **文件命名格式**: `{公司名称}_{股票代码}_投资分析报告.md`
3. **正确示例**:
   - `腾讯控股_00700_投资分析报告.md`
   - `阿里巴巴_09988_投资分析报告.md`
   - `美团_03690_投资分析报告.md`

4. **禁止事项**:
   - ❌ 不要创建 "快速参考"、"速览" 等辅助文件
   - ❌ 不要使用非标准命名格式
   - ❌ 不要输出到其他目录
